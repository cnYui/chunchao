# 第二页谱面粒子触碰效果

## 目标

- 在第二页全屏背景上保留琴谱叠加层
- 使用现有 MediaPipe 手部识别结果，让用户手碰到琴谱时，局部像粒子一样散开
- 手移开后，粒子自然回弹并重新聚合成琴谱

## 当前实现

- 第二页的谱面不再直接以静态图片显示
- 前端会读取 `pic/琴谱 new_transparent.png`
- 按透明谱面图的黑色区域采样生成粒子
- 粒子绘制在 `score-sheet` 内部的 canvas 上
- 当粒子系统准备完成后，原始谱面 `img` 透明，仅保留粒子画布可见
- 左上角调参面板当前已重新暴露粒子相关参数，方便现场直接校正粒子效果

## 手部驱动方式

- 继续复用现有的 MediaPipe Hand Landmarker
- 第 1 页乐器交互仍然使用掌心中心作为主要接近源
- 第 2 页粒子效果改为使用多关键点作为排斥源：
  - wrist
  - thumb tip
  - index tip
  - middle tip
  - ring tip
  - pinky tip

原因：

- 这样更接近“手触碰到琴谱某一部分”的感受
- 比只用掌心中心更容易形成局部散开的效果

## 粒子行为

- 粒子默认停留在各自的原始谱面位置
- 当手部关键点接近时，粒子会沿远离触碰点的方向被推开
- 推开过程中加入少量切向扰动，让散开更像“碎粒飞散”，而不是僵硬直线位移
- 没有触碰时，粒子通过弹簧回拉和阻尼回到原位

## 取舍

- 没有引入 WebGL 或额外粒子库，继续保持纯静态页面方案
- 粒子是按谱面图采样生成，不是全屏粒子场
- 第二页原先残留的拖拽谱面逻辑已经移除，避免和 MediaPipe 触碰效果冲突
- 当前保留鼠标作为第二页粒子触碰的后备输入，并补充了点击即触发的路径，原因是便于桌面环境快速测试
