# 春潮乐器交互场景

## 目标

用现有 `pic/` 素材做一个可直接在浏览器呈现的排练室场景。

核心交互：

- 鼠标接近乐器时，乐器沿远离鼠标的方向产生斥力位移
- 鼠标移开后，乐器回到初始锚点

## 当前实现

- 采用最小静态方案：`index.html` + `styles.css` + `main.js`
- 不引入框架，原因是当前项目只有素材，没有既有前端工程，直接静态实现成本最低
- 背景图当前使用 `pic/bg1.png`
- 新增一个备用背景 `pic/bg2.png`
- 当前谱面素材使用 `pic/score-transparent.png`
- 乐器图层使用透明背景版本：`pic/violin_transparent.png`、`pic/shuqin_transparent.png`、`pic/dianziqin_transparent.png`、`pic/jiazigu_transparent.png`、`pic/guitar_transparent.png`
- 页面为纯场景模式，只保留背景图、乐器贴图和鼠标排斥效果
- 已删除拖拽摆位、坐标导出、`localStorage` 测试逻辑和实时滑杆面板
- 页面不保留任何标题、说明、提示或调参文字
- 原背景文件已重命名为 `pic/bg1.png`
- 场景容器按新背景比例 `1774:887` 在视口内等比完整展示
- 当前页面已改为双页结构：第一页为互动场景，第二页为谱面展示页
- 第二页背景图 `pic/bg2.png` 当前按完整显示处理，未被裁切，剩余区域填充纯黑
- 临时摆位、缩放与坐标导出工具已移除

## 已确认默认摆位

- 提琴：`x=62.5`，`y=50.3`，`size=20`
- 竖琴：`x=82.3`，`y=53.3`，`size=16.9`
- 电子琴：`x=49.5`，`y=56.2`，`size=13.9`
- 架子鼓：`x=27.9`，`y=55.5`，`size=20.2`
- 吉他：`x=72.3`，`y=48.5`，`size=18.7`

## 已确认默认排斥参数

- `radiusBase=0.4`
- `sizeScale=2`
- `impulse=0.25`
- `spring=0.04`
- `damping=0.95`
- `xLimit=0.32`
- `yLimit=0.36`
- 单乐器参数保持当前 `data-influence` / `data-force`

## 交互设计

- 每个乐器有一个固定锚点，使用背景场景上的百分比坐标定位
- 每个乐器的默认展示大小也已固定在源码的 `data-size`
- 鼠标进入乐器影响半径后，按距离衰减计算斥力
- 位移不是直接跳变，而是通过速度 + 阻尼 + 弹簧回拉来形成连续运动
- 鼠标离开场景后，目标位移归零，乐器自然回位
- 场景全屏铺满浏览器视口，不显示额外 UI

## 取舍

- 先用中心点近似乐器碰撞体，而不是逐像素命中
  - 原因：当前目标是快速建立整体动效，逐像素命中复杂度高且收益有限
- 先手工布置初始位置，而不是做自动排版
  - 原因：素材是插画风格，视觉构图比纯算法摆放更重要
- 页面支持移动端打开，但交互重点仍然是桌面端鼠标体验

## 后续可扩展

- 为不同乐器增加不同质量、阻尼和最大位移，形成更细的性格差异
- 接入 hover 音效或环境声
- 在回位阶段加入微小旋转，强化“被惊动后回弹”的感觉
