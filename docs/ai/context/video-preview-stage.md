# 视频时间轴阶段接入

## 当前目标

- 先在前端默认展示参考视频
- 把两个现有交互页面插入到视频时间轴中，而不是一直隐藏
- 给交互页补“继续”按钮，让用户主动结束当前交互并恢复后续视频
- 给视频层补一个“跳过”按钮，让用户直接跳到下一个交互节点或最终展示页
- 把视频中的指定时间段整体替换成交互，而不是只在命中点暂停

## 当前实现

- 页面默认模式切到 `video-timeline`
- 当前前端实际播放为 3 个独立视频：
  - `video/1a21117e4e7916df5b51a3864ea114a9_raw.mp4`
  - `video/4577a95e9284af02d27603fb8d11bc3e_raw.mp4`
  - `video/e1e4aa1ee14b1794a6d6d781f966be1a_raw.mp4`
- 第 3 个视频自然结束后，前端会显示最终页面 `synthesizer.html`
- 旧的 `part1 / part2 / part3` 物理分段文件和旧参考视频已从 `video/` 删除，当前仓库不再保留未使用视频
- 三段视频与两个交互插入点统一维护在 `timeline-config.js`
- 视频为保证刷新后自动播放，初始仍使用静音自动播放
- 视频层右下角增加音量控件，用户点击后可解除静音并调节音量
- 视频层已移除底部控制条
- 视频播放区右上角增加“跳过”按钮
- 原有的：
  - 第 1 页乐器交互场景
  - 第 2 页谱面粒子场景
  - 右下角手部识别预览
  当前不删除 DOM 和逻辑代码，而是通过时间轴按需显示
- 当前顺序配置：
  - 片段 1：`1a21117e4e7916df5b51a3864ea114a9_raw.mp4`
  - 交互 1：第 2 页琴谱页
  - 片段 2：`4577a95e9284af02d27603fb8d11bc3e_raw.mp4`
  - 交互 2：第 1 页 `bg1`
  - 片段 3：`e1e4aa1ee14b1794a6d6d781f966be1a_raw.mp4`
  - 最终展示：`synthesizer.html`
- “跳过”当前行为：
  - 片段 1 播放中点击“跳过”，直接进入第 2 页琴谱页
  - 片段 2 播放中点击“跳过”，直接进入 `bg1`
  - 片段 3 播放中点击“跳过”，直接进入最终展示页 `synthesizer.html`
  - 最终展示页出现后，按钮自动隐藏
- 当前视频时间轴模式会把刷新恢复点写入 `sessionStorage`
- 当前刷新恢复语义为：
  - 若刷新时在片段视频阶段，恢复到当前片段开头
  - 若刷新时在交互页，恢复到当前交互页，不回到片段 1
  - 若刷新时在最终展示页，恢复到最终展示页
- 交互出现时：
  - 视频暂停
  - 交互层覆盖在视频上方
  - 右上角显示“继续”按钮
  - “继续”始终按当前视频片段顺序进入下一个物理分段，不再依赖交互索引反推
  - 第一个交互点击“继续”后切到片段 2
  - 第二个交互点击“继续”后切到片段 3
  - 右下角识别预览窗口需压在交互舞台之上
  - 第 2 页琴谱容器需压在谱面场景内容之上，但不遮挡“继续”按钮

## 取舍

- 继续复用现有两页交互，不重做新页面，原因是当前目标只是把交互编排进视频节奏
- `synthesizer.html` 作为独立页面通过 `iframe` 加载，原因是它自带 Tailwind CDN、字体和交互脚本，隔离后不会污染当前主页面样式
- 视频层去掉手动控件，原因是当前装置更接近固定编排而不是通用播放器
- 视频层初始静音自动播放，原因是浏览器默认会拦截带声音的刷新后自动播放；声音交给用户点击右下角音量控件后开启
- 改为物理分片播放，原因是 `currentTime` 跳转在交互恢复时容易回到整段视频开头
- 交互层只在时间轴命中后出现，平时不抢视频操作
- 刷新恢复只记录“时间轴节点”，不记录视频具体秒数，原因是当前需求是停留在当前交互页或当前片段，而不是做任意时刻断点续播

## 当前验证

- 已在 `http://127.0.0.1:4173/` 实测：
  - 初始加载播放 `part1`
  - 第一个交互点击“继续”后切到 `part2`
  - 第二个交互点击“继续”后切到 `part3`
  - 进入 `part3` 后右上角“跳过”继续显示，可直接进入最终展示页
  - 第一个交互页的谱面在首次进入时已能直接显示，不再需要先触发一次窗口缩放

## 问题记忆

- 谱面粒子初始化不能发生在交互页隐藏阶段，原因是隐藏时 `scoreSheet.clientWidth / clientHeight` 会得到 `0`
- 若在隐藏阶段把粒子画布尺寸钳制成 `1px`，首次进入交互时只会留下极小采样结果，看起来像“什么都没显示”
- 当前修复方式是：
  - 隐藏态不再接受 `0 -> 1px` 的伪尺寸
  - 时间轴切入交互页后下一帧重新执行一次 `rebuildScoreParticles()`
