- `pic/琴谱 new_transparent.png`

- 双页前端结构

- 第 2 页的琴谱完整调参面板

## 当前页面定义

### 第 1 页

- 继续使用 `pic/bg1.png`
- 保留乐器贴图、手部输入和斥力交互
- 这是当前主要互动页

### 第 2 页

- 背景改为 `pic/score-bg-fullscreen.png`
- 背景按全屏铺满方式展示
- 中央展示 `pic/琴谱 new_transparent.png`
- 当前不承载乐器斥力交互
- 当前已升级为谱面粒子页，不再是纯静态谱面图
- 琴谱当前恢复为可调 `x / y / width / height / scale / stretchX / stretchY`
- 左上角调参面板同时保留粒子参数控制
- 左上角调参面板可直接复制当前 JSON，供后续固化源码

## 已确认默认琴谱参数

```json
{
  "x": 47.8,
  "y": 32.2,
  "width": 83,
  "height": 140.1,
  "scale": 0.53,
  "stretchX": 1.06,
  "stretchY": 0.81
}
```

## 交互规则

- 左下角按钮负责前后翻页
- 键盘左右方向键也可以翻页
- 当不在第 1 页时，乐器接近交互暂停
- 第 2 页复用现有 MediaPipe 手部识别结果驱动谱面粒子散开和回收
- 第 2 页保留鼠标移动和点击作为后备触碰输入，方便桌面环境调试
- 当前先用调参面板确定新背景下的琴谱摆位，待用户回传参数后再收回调参 UI
- “只上下拉伸”只改变谱面高度，不改宽度
- “只左右拉伸”只改变谱面宽度，不改高度

## 设计原因

- 第 1 页的乐器页和第 2 页的谱面页仍然分离，避免背景替换牵连乐器摆位
- 新背景的构图和旧 `bg2` 不同，旧参数没有继续复用价值，所以先恢复摆位流程
- 摆位参数继续保留 `width / height / scale`，并补 `stretchX / stretchY`，原因是用户需要分别控制基础尺寸、整体缩放和单轴拉伸
- 粒子参数重新放回同一面板，原因是用户需要同步校正素材形状和粒子手感
- 先把调参和复制出口放回前端，用户确认后再固化为最终展示版
