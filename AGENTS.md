# 项目记忆

## 当前前端方向

- 当前项目以静态浏览器场景为起点，不先引入框架
- 原因：现阶段只有图片素材，目标是尽快验证交互概念

## 已确认的交互目标

- 使用 `pic/` 下的背景图和乐器贴图构建排练室场景
- 用户鼠标接近乐器时触发斥力效果
- 用户鼠标移开后，乐器恢复到原始位置
- 当前新的装置目标是把“鼠标接近”升级为“真实手部接近”
- 左手或右手都可以作为交互输入
- 若目标是“手真的靠近墙上的乐器才触发”，后续必须补充侧拍手机通道做距离估算

## 当前实现约束

- 优先复用现有素材，不新增外部依赖
- 先解决核心交互，再考虑音效、剧情或更复杂编排
- 如果后续需要更复杂的时间线、音频同步或多场景切换，再评估是否升级为正式前端工程
- 默认摆位以用户最终确认的导出坐标为准，并同步更新源码与上下文文档
- 当前页面输出以纯场景展示为准，不保留说明文字或调参 UI
- 当前乐器素材来源以 `pic/violin_transparent.png`、`pic/shuqin_transparent.png`、`pic/dianziqin_transparent.png`、`pic/jiazigu_transparent.png`、`pic/guitar_transparent.png` 为准
- 当前背景素材以 `pic/bg1.png` 为准
- 当前已新增一个备用背景 `pic/bg2.png`
- 当前谱面素材以 `pic/score-transparent.png` 为准
- 当前视频素材统一放在根目录 `video/`
- 当前已添加参考视频 `video/33bf26e484a3a504099f10ecbd2e8c13.mp4`
- 当前已从原始参考视频物理裁出三段：`video/33bf26e484a3a504099f10ecbd2e8c13_part1.mp4`、`video/33bf26e484a3a504099f10ecbd2e8c13_part2.mp4`、`video/33bf26e484a3a504099f10ecbd2e8c13_part3.mp4`
- 当前 `video/33bf26e484a3a504099f10ecbd2e8c13_part2.mp4` 已额外删除开头 1 秒内容，当前对应原视频区间为 `44s-51s`
- 若更换背景导致构图变化，先恢复临时摆位流程，让用户拖拽确认新坐标，再固化为默认值
- 当前默认摆位已包含用户确认的 `x / y / size`，后续若再调整需一起回写三项
- 当前页面已回到最终展示态，不保留临时摆位和缩放工具

## 素材处理记忆

- `pic/` 下 5 张乐器图已生成透明背景版本，文件名为 `*_transparent.png`
- 后续页面集成乐器素材时优先使用透明版，避免背景底色或假透明棋盘格覆盖场景
- 透明抠除采用“边缘连通区域去背景”策略，不直接全图删浅色，原因是要保留乐器内部高光和留白
- 如果输入是 `jpg`，透明结果一律导出为 `png`，不覆盖原始 `jpg`
- 对假透明棋盘格素材，若主体内部仍残留背景孤岛，需要追加第二轮按连通块清理

## 交互方案记忆

- 当前把用户提到的 `Media Pipeline` 按 `MediaPipe` 理解
- 第一阶段优先做前视单机 MVP：电脑摄像头 + MediaPipe Hand Landmarker，把手部二维位置映射成乐器斥力输入
- 第一阶段先解决“画面内接近”，不强行解决真实墙面深度
- MediaPipe 的手部 `z` 和 `world landmarks` 不能直接当作“手到墙面距离”
- 第二阶段若要实现真实空间接近，使用侧拍手机估算手到墙面的距离，并通过局域网把数据回传到投影电脑页面
- 第二阶段若启用手机侧拍通道，需要引入最小通信层，不能假设纯静态页面自己充当 WebSocket 服务端
- 第一阶段交互点优先使用掌心中心，不优先使用指尖，原因是掌心抖动更小、更适合作为接近源
- 第一阶段已在 `main.js` 中把输入层抽象为统一交互源，鼠标保留为后备输入
- 第一阶段当前通过官方 CDN 动态加载 `@mediapipe/tasks-vision`，原因是项目仍保持静态页形态、先验证交互
- 当前手部输入运行依赖安全上下文，需在 `localhost` 或 `https` 下打开页面，`file://` 不能启用摄像头
- 当前页面右下角保留一个小型摄像头预览窗口，用于现场确认手部关节点是否被识别到
- 当前右下角识别预览窗口需要压在交互舞台之上，避免被场景层级盖住
- 当前页面左下角保留翻页控件，共两页
- 第 1 页继续承载乐器互动，第 2 页使用 `pic/bg2.png` 作为背景并展示 `pic/score-transparent.png`
- 第 2 页背景当前必须完整显示 `bg2`，不允许裁切，剩余区域保持纯黑
- 第 2 页琴谱调试界面已删除，当前页面回到展示态
- 第 2 页当前已改为谱面粒子页，手部触碰时局部粒子散开，手移开后回收重组
- 第 2 页粒子效果复用现有 MediaPipe 手部追踪，并使用多关键点而不是单一掌心点
- 第 2 页原先残留的拖拽谱面逻辑已移除，避免和粒子触碰效果冲突
- 第 2 页当前保留鼠标移动和点击作为粒子触碰后备输入，原因是现场调试时不一定每次都先开摄像头
- 第 2 页当前默认琴谱参数已确认：`x=46.6`、`y=18.7`、`width=83`、`height=140.1`、`scale=0.3`
- 第 2 页琴谱容器当前需保持在谱面交互内容最上层，但不覆盖右上角“继续”按钮
- 当前时间轴模式下，谱面粒子必须在交互页真正显示后再重建一次，原因是隐藏态尺寸为 `0`，否则首次进入会看起来像未显示
- 当前已新增视频播放器层，使用 `video/33bf26e484a3a504099f10ecbd2e8c13.mp4` 作为主内容
- 当前视频层不再允许用户通过前端直接暂停、播放或拖动进度，原因是播放节奏要让位于时间轴编排
- 当前视频层在交互出现时会让出舞台给交互覆盖层，避免视频按钮和交互层同时抢操作
- 当前视频为保证刷新后自动播放，前端固定使用静音自动播放模式
- 当前视频时间线不再依赖 `currentTime` 跳转，原因是“继续”后需要稳定进入下一段视频而不是回到整段开头
